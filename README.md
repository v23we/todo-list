# TodoList

一个原生 iPhone 的中文轻游戏化待办 App MVP，技术栈固定为 `Swift + SwiftUI + SwiftData`，目标是把首页注意力收窄到“现在先做什么”，并用温和的成长反馈替代压力打卡。

## 运行方式

1. 用 Xcode 26.3 或更高版本打开 [TodoList.xcodeproj](/Users/lcy/Code/todo-list/TodoList.xcodeproj)
2. 选择 `TodoList` scheme
3. 选择 iPhone 模拟器并运行

命令行构建示例：

```bash
xcodebuild -project TodoList.xcodeproj -scheme TodoList -destination 'platform=iOS Simulator,name=iPhone 16' build
```

## 当前功能

- 首页固定展示 `1 个当前任务 + 2 个备选任务`
- 首页只显示任务标题、下一步和总进度；有子任务时“下一步”自动等于第一个未完成子任务
- 任务详情页支持完整展开子任务、逐个勾选、自动推进父任务完成
- 完成任务固定 `+20 XP`
- “稍后”任务不会进入首页当前任务或备选任务，可随时移回待处理
- 经验条增长、升级、奖励解锁主链路完整
- 奖励页可切换主题、XP 样式、音效占位、轻量完成动效
- 设置页可切换音效、震动、减少动态效果
- 所有数据本地持久化

## 核心实现说明

- 任务经验值在 P0 固定为 `+20 XP`，不提供自由编辑入口
- 当前任务补位规则：
  - 先使用首页当前显示顺序中的备选任务补位
  - 备选不足时，从 `待处理` 中按 `sortOrder -> createdAt` 补齐
  - `稍后` 分区不参与首页补位
  - 若没有可补位任务，首页显示空状态
- “下一步”规则：
  - 没有子任务时，首页展示手动填写的“下一步”
  - 有子任务时，首页展示第一个未完成子任务，手动“下一步”不再作为事实源
  - 创建页/详情页会在子任务区域补一条轻提示，说明首页“下一步”会自动取第一个未完成子任务
- 完成规则：
  - 没有子任务时，点击“做完”直接完成父任务
  - 有子任务时，首页按钮改为“完成这一步”，每次只完成当前第一个未完成子任务
  - 所有子任务完成后，父任务才自动完成并结算 XP / 升级 / 补位
- XP 样式切换会真实影响完成反馈图标；当前支持 `小闪电 / 小液滴 / 小咖啡豆`
- 完成动效目前提供轻量差异：柔光反馈与轻波纹
